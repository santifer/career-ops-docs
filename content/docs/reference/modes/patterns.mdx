## What it does

patterns reads every report in the rejected and discarded stages, looks for common features (industry, company size, seniority level, specific keywords), and produces a structured analysis of what is and is not working in your targeting.

## When to use it

Run patterns after every 20–30 rejections to recalibrate. The mode surfaces blind spots in your profile and helps you avoid sending applications into the same dead patterns.

## Example

```
`/career-ops patterns` after a month of search — output is a structured pattern analysis with recommendations to adjust config/profile.yml or your scan filters.
```

## Gotchas

patterns needs a meaningful sample size (15+ rejections) to surface real signal. With fewer rejections, the analysis is noise.
